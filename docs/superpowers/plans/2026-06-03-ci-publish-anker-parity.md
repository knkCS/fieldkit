# CI/Publish Anker Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring `@knkcs/fieldkit`'s CI and publish GitHub Actions to parity with `@knkcs/anker` — add an export-surface guard, run full checks before publishing, publish to both npmjs.org (with provenance) and GitHub Packages, and switch the release tag scheme to `v*`.

**Architecture:** Port anker's repo-agnostic `verify-exports.ts` script (parses `tsup.config.ts` entries, diffs source exports against built `.d.ts`), wire it into `package.json` and CI, then rework the publish workflow to gate on full checks and dual-publish. No application/runtime code changes.

**Tech Stack:** GitHub Actions, npm, tsup, tsx, TypeScript compiler API (used by the ported script).

**Reference spec:** `docs/superpowers/specs/2026-06-03-ci-publish-anker-parity-design.md`

---

## File Structure

- `scripts/verify-exports.ts` — **new**, ported verbatim from anker. Self-contained Node/tsx script; the only new source file. One responsibility: assert every entry's source export surfaces in its built `.d.ts`.
- `package.json` — **modify**. Add `tsx` devDep + `verify-exports` script.
- `.github/workflows/ci.yml` — **modify**. Add one `verify-exports` step.
- `.github/workflows/publish-fieldkit.yml` — **modify**. New trigger, permissions, full check gate, dual publish.
- `package-lock.json` — **regenerated** by `npm install` when `tsx` is added.

## Task 3: Rework the publish workflow for anker parity + dual publish

**Files:**
- Modify: `.github/workflows/publish-fieldkit.yml`

- [ ] **Step 1: Replace the workflow file contents**

Overwrite `.github/workflows/publish-fieldkit.yml` with:

```yaml
name: Publish @knkcs/fieldkit

on:
  push:
    tags:
      - "v*"

permissions:
  contents: read
  id-token: write
  packages: write

jobs:
  publish:
    name: Publish to npmjs.org and GitHub Packages
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          registry-url: https://registry.npmjs.org
          cache: npm
          cache-dependency-path: package-lock.json

      - name: Install deps
        run: npm ci

      - name: Verify version matches tag
        run: |
          PKG_VERSION=$(node -p "require('./package.json').version")
          TAG_VERSION="${GITHUB_REF_NAME#v}"
          if [ "$PKG_VERSION" != "$TAG_VERSION" ]; then
            echo "package.json version ($PKG_VERSION) does not match tag ($TAG_VERSION)"
            exit 1
          fi

      - name: Lint
        run: npm run lint

      - name: Typecheck
        run: npm run typecheck

      - name: Build
        run: npm run build

      - name: Verify exports
        run: npm run verify-exports

      - name: Test
        run: npm test

      - name: Publish to npmjs.org
        run: npm publish --provenance --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          registry-url: https://npm.pkg.github.com
          scope: "@knkcs"

      - name: Publish to GitHub Packages
        run: npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
